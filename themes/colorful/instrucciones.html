Instrucciones de instalación

Subir las páginas y los formularios

La manera más rápida de importar las plantillas de las páginas y los bloques de código (formularios) es utilizar el plugin mcw_templates ya que yo las he exportado haciendo uso del mismo. Una vez hecha la importación se puede desactivar e incluso borrar el mismo.

Para ello en tu directorio textpattern debes crear el directorio _templates y debe contener a su vez un directo<rio con archivos clasificados por subdirectorios listos para importar. Este método no me ha funcionado para subir el CSS.

Es un medio seguro, ya que la hacer la importación hará una copia de seguridad de tus plantillas previas.

Subir el CSS

Abrir el fichero que contiene el CSS y copiarlo sobre la caja de texto en Presentación —> Estilos. Si se desea guardar con un nombre que no sea default se puede crear una nueva, pero luego tienes que acordarte en el momento de configurar las secciones.

Subir las imágenes y los archivos

Este tema sólo hace uso de dos imágenes. Para subirlas haz uso de las opciones del propio Textpattern (Contenido —> Imágenes).

Para subir el favicon usa también la interfaz de Textpattern (Contenido —> Archivos). Ponle como título favicon.ico y una descripcción del estilo “Favourite icon”.

Editar el CSS y las plantillas con las rutas correctas de las imágenes

Busca la línea background: #CCCCCC url(/colorful/images/3.png) repeat top left; en el CSS y cambia la ruta de la imagen y el nombre de la misma por el que se corresponda a tu instalación. Lo puedes ver en el menú de imágenes.

En la plantilla navigation tienes que cambiar la imagen que aparece en la línea <li><txp:feed_link flavor="rss" section="articulos"><txp:image id="5"/></txp:feed_link></li> por el identificador de la imagen para RSS de tu instalación. Solo cambiar el número que aparece en id="5" por el que corresponda.

Editar el formulario DTD_head

Edita el formulario DTD_head.

Busca la línea <link rel="shortcut icon" href="<txp:site_url />files/colorful.ico" cambiando files/colorful.ico por la ruta relativa hasta el archivo .ico que has subido a tu sitio. Normalmente sólo tienes que cambiar algo si le has cambiado el nombre al fichero .ico.

Configurar las secciones

Las secciones tienen que quedar de la siguiente manera:

Sección Página Uso
archivo-mensual archive Archivos mensuales
archivos-por-categoria archive Archivos por categoría
articulos archive Mostrar los artículos
buscar default Búsqueda
contacto default Página de contacto
info default Página “Acerca de”
palabraclave default Búsqueda por etiqueta

Todas las secciones utilizan la plantilla CSS default aunque si se desea que alguna utilice otro estilo se puede crear otra hoja de estilo y asignarla.

Crear un artículo de contacto

  1. Crea un artículo con el nombre que desees.
  2. Asignarle la sección contacto
  3. El artículo debe contener <txp:output_form form="email /> en el lugar en el que desees mostrar el formulario de contacto.

Crear un artículo “acerca de” p.

  1. Crea un artículo con el nombre que desees
  2. Asignarle la sección info
  3. Escribe el texto típico “acerca de” que consideres oportuno.

Configurar plugins:

En ests sitio de muestra están instalados los siguientes plugins:

NameAuthorVersionDescription
mcw_templatesMike West0.2File Based Templates
glx_gravatarDavid's Neighbour2011.1Implementation of gravatar for Textpattern.
mkp_versionMichael K Pate1.0Displays the current version number
tru_tagsNathan Arthur3.6Article tagging
zem_contactThreshold State0.6Flexible email contact/feedback form
rsx_plugins_tableRamanan Sivaranjan1.2Display a list of plugins currently installed.

De estos plugins mcw_templates se puede desactivar o incluso eliminar después de la importación de los formularios, glx_gravatar es necesario para mostrar los avatares en los comentarios, mkp_version no es necesario y no se incluye en el paquete de instalación, tru_tags es necesario para la navegación por palabras clave, zem_contact se utiliza para gestionar el formulario de contacto, aunque se puede sustituir por zem_contact_reborn y rsx_plugins_table es innecesario, sólo se utiliza para mostrar la lista de plugins, por lo que tampoco se incluye en la instalación.

El único plugin que necesita configuración de todos estos es tru_tags y esa configuración consiste en ir a Extensiones —> tru_tags y en el campo que tiene como título Tag section name (default is “tag”): poner palabraclave

En serio, esto es todo. No es una instalación en un sólo clic pero siguiendo estos pasos no te puedes equivocar. Si tienes alguna duda o te quedas atascado en alguno de los pasos no dudes en contactar conmigo.
